## 🔐 Настройка API Rules (PocketBase)

Для правильной работы ролей (Админ/Сотрудник), пропиши следующие правила в Admin UI PocketBase:

### Коллекция `equipment`
| Действие | Правило |
|----------|---------|
| **List/View** | `@request.auth.id != ""` |
| **Create/Update/Delete** | `@request.auth.role = "admin"` |

### Коллекция `usage`
| Действие | Правило |
|----------|---------|
| **List/View** | `@request.auth.role = "admin" || @request.auth.id = used_by` |
| **Create** | `@request.auth.id != ""` |
| **Delete** | `@request.auth.role = "admin"` |

## 🌐 Развёртывание (Coolify)

1. Создайте новый **Static Site** в Coolify.
2. Подключите ваш GitHub репозиторий.
3. Убедитесь, что `tester.html` доступен по прямому URL или переименован в `index.html` для автоматического запуска.

## 🐛 Решение проблем

- **400 Error:** Проверьте длину пароля (должно быть 8+ символов) и уникальность Email.
- **403 Forbidden:** Проверьте API Rules (см. таблицу выше) и убедитесь, что вы залогинены.
- **Mixed Content:** Если ваш PocketBase на HTTP, а сайт на HTTPS, запросы могут блокироваться браузером.
